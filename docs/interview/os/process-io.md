# 操作系统：进程线程与 IO 模型

## 进程 vs 线程 vs 协程

| 维度 | 进程 | 线程 | 协程 |
|------|------|------|------|
| 资源拥有 | 独立地址空间、文件描述符 | 共享进程资源 | 共享线程资源 |
| 切换开销 | 高（TLB 刷新、上下文切换） | 中（寄存器保存/恢复） | 低（用户态切换） |
| 通信方式 | IPC（管道/共享内存/信号） | 直接读写共享内存 | 共享内存 + channel |
| 创建开销 | 高（fork 复制页表） | 中 | 极低 |

## IPC 方式

1. **管道 (pipe)**: 父子进程间单向通信
2. **命名管道 (FIFO)**: 无亲缘关系进程间通信
3. **共享内存 (shm)**: 最快的 IPC，需同步机制配合
4. **消息队列 (msgqueue)**: 内核维护的链表
5. **信号量 (semaphore)**: 同步原语
6. **信号 (signal)**: 异步事件通知
7. **Socket**: 跨网络通信

## 虚拟内存 & 页表

- 虚拟地址 → MMU → 页表 → 物理地址
- 多级页表: 4 级分页（PGD/PUD/PMD/PTE），减少页表占用
- TLB: 页表的高速缓存，上下文切换时需刷新（CR3 寄存器切换）

## 页面置换算法

| 算法 | 思路 | 优劣 |
|------|------|------|
| FIFO | 先进先出 | 实现简单，Belady 异常 |
| LRU | 最近最久未使用 | 近似最优，实现开销高 |
| Clock | 第二次机会 | LRU 的近似，软硬件协同 |
| LFU | 最不经常使用 | 热点数据友好，历史影响大 |

## IO 模型

1. **阻塞 IO**: 系统调用直到数据就绪才返回
2. **非阻塞 IO**: 立即返回，轮询检查
3. **IO 多路复用**: 单线程监听多个 fd
4. **信号驱动 IO**: SIGIO 通知
5. **异步 IO (AIO)**: 内核完成操作后通知

## select / poll / epoll

| 特性 | select | poll | epoll |
|------|--------|------|-------|
| fd 上限 | 1024 (FD_SETSIZE) | 无限制 | 无限制 |
| 数据结构 | bitmap | pollfd 数组 | 红黑树 + 链表 |
| 监听注册 | 无 | 无 | epoll_ctl 注册 |
| 就绪通知 | 遍历所有 fd O(N) | 遍历所有 fd O(N) | 回调 + 只返回就绪 fd O(1) |
| 工作模式 | LT | LT | LT + ET |

## 零拷贝

- **sendfile**: 内核空间直接传数据，无需经过用户空间
- **splice**: 在两个 fd 之间移动数据
- **mmap**: 文件映射到进程地址空间，减少一次 copy

## 实操总结

- `strace -p <pid>` 追踪系统调用，观察 IO 模型
- 高并发服务 epoll 是标配，配合非阻塞 IO + 事件循环（如 Redis、Nginx）
- 线程池大小参考公式: CPU 密集型 = N+1，IO 密集型 = 2N
